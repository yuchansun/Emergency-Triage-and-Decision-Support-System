# 學校 Windows 電腦 — 7B / 14B 自動化評測指南

> 給 Cursor 開著這份文件，**由上往下逐步做**即可。  
> 評測腳本：`eval/evaluate_models.py`  
> 測試資料：`eval/test_cases.jsonl`（64 筆）

---

## 該用 Mac 還是學校電腦？

| | MacBook Air **M4 16GB** | 學校桌機 **i7-10700 16GB**（Intel UHD 630） |
|---|---|---|
| LLM 推理方式 | **Metal GPU 加速**（快） | 多半 **純 CPU**（無獨立顯卡，慢很多） |
| 7B 64 筆粗估 | **~1 小時**（你之前實測 ~57 秒/筆） | **~2～4 小時** |
| 14B 64 筆粗估 | **~2～3 小時**（但會燙、耗電） | **~4～8 小時** |
| 散熱 / 供電 | 無風扇，長跑易降頻；充電可能跟不上 | 有風扇、插電穩，適合放著跑 |
| 中斷續跑 | ✅ 支援（`--label` 相同會自動跳過已完成） | ✅ 同上 |

**結論**

- **比速度**：你的 **M4 Mac 明顯比較快**（學校那台沒有 NVIDIA 顯卡，Ollama 幾乎靠 CPU 跑）。
- **比穩定長跑 14B**：學校桌機較穩（不會像 Air 那樣燙到降頻或掉電關機）。
- **實務建議**：
  - 時間有限、想快 → **Mac 分批跑**（見下方「Mac 替代方案」）。
  - 學校能待一整天、Mac 14B 老是被中斷 → **用學校電腦慢慢跑完**，反正有續跑，關機也不怕。

## 第一部分：學校電腦第一次設定（只做一次）

> 需要能安裝軟體（Ollama、Python、XAMPP）。若學校擋安裝，請先問管理員。

### 1. 安裝軟體

| 軟體 | 下載 | 備註 |
|------|------|------|
| Ollama | https://ollama.com/download | 裝完通常常駐背景 |
| Python 3.10+ | https://www.python.org/downloads/ | 安裝時勾 **Add python.exe to PATH** |
| XAMPP | https://www.apachefriends.org/ | 只用 MySQL |

### 2. 下載 LLM 模型

開 **PowerShell**，執行（需網路，約 15～40 分鐘）：

```powershell
ollama pull qwen2.5:7b-instruct
ollama pull qwen2.5:14b-instruct
```

### 3. 匯入資料庫

1. 開 XAMPP → 按 **Start** 啟動 **MySQL**
2. 瀏覽器開 http://localhost/phpmyadmin
3. 建立資料庫：`medical_triage_system`
4. 選該庫 → **匯入** → 選隨身碟的 `medical_triage_system.sql`

### 4. 建立 Python 環境

以下假設專案在 `D:\triage_project\專題-react-version`，請依實際路徑修改。

```powershell
cd D:\triage_project\專題-react-version\llm-backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
pip install fastapi uvicorn pymysql python-dotenv pandas
```

> 第一次啟動 LLM 後端時，還會自動下載嵌入模型（幾百 MB），屬正常現象。

### 5. 確認 `.env`

編輯 `llm-backend\.env`，至少要有：

```env
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:7b-instruct
OLLAMA_NUM_PREDICT=200
OLLAMA_TIMEOUT=30
```

## 第二部分：每次跑評測（開 4 個 PowerShell 視窗）

### 視窗 A — Ollama（若已常駐可跳過）

```powershell
ollama serve
```

### 視窗 B — 主後端（port 8000）

```powershell
cd D:\triage_project\專題-react-version\api
python -m uvicorn main:app --port 8000
```

✅ 確認：瀏覽器開 http://localhost:8000/triage_hierarchy 有 JSON 資料。

### 視窗 C — LLM 後端（port 8001）

```powershell
cd D:\triage_project\專題-react-version\llm-backend
venv\Scripts\activate
python -m uvicorn main:app --port 8001
```

✅ 等到出現這兩行才算 ready：

```
✅ Ollama 設定：... model=qwen2.5:7b-instruct ...
INFO:     Application startup complete.
```

### 視窗 D — 評測腳本

#### 步驟 0：試跑 3 筆（正式跑前必做）

```powershell
cd D:\triage_project\專題-react-version\llm-backend
venv\Scripts\activate
python eval/evaluate_models.py --label test-run --limit 3 --fresh
```

3 筆都成功、沒有 error → 繼續。

#### 步驟 1：跑 7B（完整 64 筆）

確認 `.env` 是 `OLLAMA_MODEL=qwen2.5:7b-instruct`，且視窗 C 已重啟並 ready。

暖機（避免第一筆逾時）：

```powershell
curl http://localhost:11434/api/generate -d "{\"model\":\"qwen2.5:7b-instruct\",\"prompt\":\"hi\",\"stream\":false}"
```

正式跑（**重跑請加 `--fresh`**）：

```powershell
python eval/evaluate_models.py --label qwen2.5-7b --fresh
```

預估時間：**2～4 小時**（學校 CPU）。每跑完一筆會自動存檔。

#### 步驟 2：跑 14B

1. 改 `llm-backend\.env` → `OLLAMA_MODEL=qwen2.5:14b-instruct`
2. **視窗 C** 按 `Ctrl+C` 關掉，再重新執行 uvicorn（同上）
3. 確認 log 顯示 `qwen2.5:14b-instruct` + `Application startup complete`
4. 暖機 14B 後執行：

```powershell
python eval/evaluate_models.py --label qwen2.5-14b --fresh
```

預估時間：**4～8 小時**（學校 CPU）。

#### 步驟 3：產生對照表

兩個模型都跑完後：

```powershell
python eval/evaluate_models.py --compare
```

輸出：`eval/results/compare.md`

## 中斷與續跑

| 情況 | 做法 |
|------|------|
| 評測跑到一半要關機 | 直接關；已完成的筆數已存檔 |
| 下次繼續 | **同一個 `--label`** 再跑同一行指令（**不要**加 `--fresh`） |
| 想全部重來 | 加 `--fresh` |

範例（7B 跑到一半被關機，下次接著跑）：

```powershell
python eval/evaluate_models.py --label qwen2.5-7b
```

## 常見問題

| 問題 | 解法 |
|------|------|
| `找不到測試資料檔` | 確認在 `llm-backend` 目錄下執行，且 `eval/test_cases.jsonl` 存在 |
| `/triage_hierarchy` 失敗 | XAMPP MySQL 有開嗎？資料庫匯入了嗎？視窗 B 有在跑嗎？ |
| 第一筆 timeout | 先暖機 curl；確認視窗 C 已 `Application startup complete` |
| 14B 記憶體不足 | 關掉 Chrome 等程式；16GB 理論上夠，但別同時開太多 |
| 路徑報錯 | 改放到 `D:\triage_project`（純英文路徑） |
| 進度卡住很久 | 學校 CPU 跑 14B 本來就慢，看 terminal 有沒有在動 |

## Mac 替代方案（若決定在自己電腦跑）

你的 **M4 比學校 CPU 快**，若選 Mac：

1. 接電源、墊高筆電通風
2. 開 Ollama + 兩個後端（同第二部分，指令改 Mac 版）
3. 分批跑，關機也不怕：

```bash
cd 專題-react-version/llm-backend
venv/bin/python eval/evaluate_models.py --label qwen2.5-7b --fresh
# 7B 跑完 → 改 .env 14B → 重啟後端 →
venv/bin/python eval/evaluate_models.py --label qwen2.5-14b --fresh
venv/bin/python eval/evaluate_models.py --compare
```

Mac 啟動後端指令：

```bash
# 視窗 B
cd 專題-react-version/api && python -m uvicorn main:app --port 8000

# 視窗 C
cd 專題-react-version/llm-backend && venv/bin/python -m uvicorn main:app --port 8001
```

## 時間規劃建議

| 階段 | 學校 i7（CPU） | Mac M4 |
|------|----------------|--------|
| 第一次設定 | 1～2 小時 | — |
| 7B × 64 筆 | 2～4 小時 | ~1 小時 |
| 14B × 64 筆 | 4～8 小時 | 2～3 小時 |
| **7B + 14B 合計** | **約 6～12 小時** | **約 3～4 小時** |

學校建議：**早到、先 pull 模型、試跑 3 筆通過再掛正式跑**；7B 跑完再換 14B，中間不用關其他視窗。
